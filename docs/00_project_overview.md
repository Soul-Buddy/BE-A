# 00. Soul Buddy — 프로젝트 전체 개요

> **AI 코딩 지침**: 이 파일은 프로젝트의 목적·범위·시나리오·팀 구성을 정의합니다.
> 무엇을 만들어야 하는지 모를 때, 범위를 벗어나려 할 때 이 파일을 먼저 확인하세요.

---

## 서비스 정의

Soul Buddy는 사용자가 작성한 개인 프로필(취미, 성격, 고민 등)을 기반으로
**매 대화마다 개인화된 컨텍스트를 자동 주입**하여 AI 상담/말벗 경험을 제공하는 서비스입니다.

### 핵심 차별점
- 사용자가 매번 배경을 설명하지 않아도 됨 (Personal Instruction 자동 주입)
- 페르소나 선택으로 같은 고민도 다른 방식으로 대화 가능
- 단순 채팅이 아닌 요약·감정 기록·회고 기능 포함
- **의료 진단 대체가 아닌 정서 지원 보조 서비스**

## MVP 기능 목록 (반드시 구현)

| # | 기능 | 담당 |
|---|------|------|
| 1 | 소셜 로그인 (OAuth) | BE-B, FE |
| 2 | 온보딩 설문 + 사용자 프로필 저장 | BE-B, BE-C, FE |
| 3 | 페르소나 선택 (친구형 / 상담사형 / 공감형) | BE-A, FE |

| 5 | 대화 종료 후 요약 생성·저장 | BE-A, BE-C |
| 6 | 감정 로그 대시보드 시각화 | BE-C, FE |
| 7 | 위험 키워드 감지 + 전문 상담 권고 | BE-A |

- 온보딩 설문 재작성 기능

## 절대 비포함 (캡스톤 범위 초과)

- 정식 의료 진단·치료 판정
- 실제 병원 예약 연동
- 쿠버네티스 운영
- 실시간 상담사 매칭

## 주요 사용자 시나리오

### 시나리오 1 — 첫 방문
1. 소셜 로그인으로 가입
2. 온보딩 설문 작성 (취미·성격·고민 유형·선호 말투)
3. 페르소나 선택 후 첫 AI 대화 세션 시작
4. 대화 종료 → 요약 카드 + 감정 로그 자동 생성

### 시나리오 2 — 재방문
1. 이전 세션 요약·감정 이력이 반영된 상태로 대화 재개
2. 대시보드에서 감정 변화 추이 확인

### 시나리오 3 — 위험 상황
1. 자해·극단적 표현 입력 감지 (riskLevel = HIGH)
2. 일반 AI 응답 차단, 안전 안내 메시지 우선 출력
3. 전문 기관 안내 문구 + 연락처 제공

## 팀 구성

| 역할 | 담당자 | 주요 범위 |
|------|--------|-----------|
| PM / 심리·데이터 | 홍록기 | 기획, 자문, 데이터 |
| Frontend | 김성환 | Next.js, UI/UX |
| Backend A (AI) | 장지훈 | LLM 연동, 프롬프트, 채팅 API |
| Backend B (인증·API) | 준영 | OAuth, 사용자 API, Swagger |
| Backend C (DB·운영) | 헌영 | DB 설계, 엔티티, Docker |
